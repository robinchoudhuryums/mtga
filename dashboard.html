Recently edited decks — push your repo changes back to Arena

The decks you've changed most recently in the repo (from git history), newest first — the repo → Arena direction. After editing a list here, copy its Arena import block and re-import it in the app so the two stay in sync. The mirror of the stale-deck check below (Arena → repo). Check for stale decks — paste your Arena export(s)

Paste one deck's Arena export to see if it drifted from the stored list, or paste several Deck blocks at once for a roster staleness report. Each block is auto-matched to its closest stored deck (variants included) — Arena exports don't carry a deck name. Compared by card name + quantity; printings and basic-land art are treated as the same card (same rules as deck.py verify). Nothing is uploaded — the compare runs entirely in your browser.

Standard rotation — what rotates next, and which decks it hits

Cards in your Standard decks whose set is closest to rotating out (release year + ~3). A wildcard on a soon-rotating card won't last — this is deck.py rotation on the dashboard. Timing is a heuristic from set release (the pool keys one printing per card, so a reprint can read early) — verify against the official schedule before disenchanting.
